## 🌌 项目简介

**《行星迁移计划 / Planetary Exodus》**  
是一个面向 **长期服务器运营** 的  
**硬科幻 · 文明级 Minecraft Java 模组**。

在这个世界中：

- 行星环境正在持续恶化  
- 恒星活动逐渐失控  
- 生态、资源与结构不断崩坏  
- 原地生存 = 缓慢灭亡  

玩家的唯一出路是：

> **推动整个文明，完成行星级迁移。**

这不是冒险模组，  
而是一场 **工程、组织与时间的考验**。

---

## 🪐 核心世界观

你所处的世界，是一颗正在走向终局的行星。

- 母恒星进入高活跃周期  
- 辐射与能量波动持续上升  
- 气候与生态系统开始失稳  

服务器不会突然毁灭世界，  
但**环境会一天天变坏**。

---

## 🔁 服务器核心循环（长期可运行）

- 资源开采  
  ↓  
- 工业体系建立  
  ↓  
- 能源与基础设施  
  ↓  
- 轨道与太空工程  
  ↓  
- 行星危机加剧  
  ↓  
- 文明级决策  
  ↓  
- 阶段性迁移  

> 每一个阶段都可以持续数周甚至数月。

---

## 🌞 行星危机系统（全服共享）

行星状态是 **服务器唯一变量**，  
影响所有玩家的生存环境。

| 状态 | 描述 | 世界影响 |
|----|----|----|
| 🟢 STABLE | 相对稳定 | 接近原版体验 |
| 🟡 STRAINED | 负荷上升 | 能源效率变化 |
| 🟠 DEGRADED | 明显恶化 | 天气异常、作物衰减 |
| 🔴 COLLAPSING | 崩溃边缘 | 辐射、结构损坏、死亡惩罚 |

行星不会“剧情杀”，  
但如果玩家什么都不做——  
它一定会走向崩溃。

---

## 🚀 行星迁移工程（不是换地图）

**行星迁移 ≠ 传送到新世界**

这是一项真正的文明工程：

- 建造轨道设施  
- 稳定近地空间  
- 转移工业与人口  
- 冻结部分地表区域  
- 激活新宜居区域 / 新星系  

迁移是 **阶段推进** 的，  
而不是一次性完成。

---

## 🛰 火箭与太空系统

火箭不是 GUI 按钮，  
而是 **真实的服务器实体**。

典型飞行阶段：

1. 🚀 发射（燃料消耗 + 风险）  
2. 🛰 轨道插入（可能失败）  
3. 🌌 推进与航行（时间推进）  
4. 🪐 抵达轨道设施 / 新区域  

采用 **阶段制 + 事件驱动**，  
保证真实感与服务器性能。

---

## 👥 玩家分工（服务器寿命核心）

一个人无法完成行星迁移。

服务器会自然形成以下角色：

- 🛠 工业工程师（生产线 / 能源）  
- 🚀 航天工程师（火箭 / 推进系统）  
- 🛰 轨道设施维护者  
- 🔬 科研人员（科技解锁）  
- 📦 后勤与物流协调者  

✔ 新手有位置  
✔ 大佬需要团队  
✔ 合作 > 数值堆叠  

---

## ☠️ 失败不是删档

如果玩家放弃推进迁移：

- 行星进入 🔴 COLLAPSING  
- 资源逐步枯竭  
- 环境持续恶化  

服务器将进入：

> **「衰亡纪元 / Decline Era」模式**

这不是 Game Over，  
而是另一条更艰难的生存路线。

---

## ⚙️ 技术实现

- Minecraft Java 1.21.1  
- Fabric  
- 单主模组 + 多子系统  
- 数据驱动配置（JSON）  

### 有意避免的内容：
- ❌ N 体模拟  
- ❌ 实时天体 / 轨道计算  
- ❌ Tick 级重物理  
- ❌ 无限制维度堆叠  

工程可控，长期可维护。
